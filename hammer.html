App Name: V2RBuilds – On-Site AI Estimate & Negotiation Assistant

1. Purpose & High-Level Summary

The app enables general contractors to record a walkthrough conversation, extract scope-of-work details, generate an itemized renovation estimate in <60 seconds, and present a polished quote to homeowners for negotiation.

The app must support multilingual contractors (Portuguese, Spanish, English) and produce a professional output instantly on-site.

2. Core User Flow (Sequential)

  1. Contractor opens the app.
  2. Presses Start Walkthrough Recording.
  3. Contractor speaks normally; app records audio.
  4. Audio is transcribed, translated, and analyzed for:
    • scope
    • materials
    • labor
    • dimensions
    • timeline
    • complexity
  5. AI generates:
    • itemized estimate
    • materials list
    • labor/timeline
    • profit margin
    • recommended quote
  6. Contractor reviews output.
  7. Contractor edits line items, materials, margins.
  8. Contractor taps Approve & Share.
  9. App generates polished PDF-style estimate.
  10. Contractor screenshares or sends estimate to homeowner.

3. Agents (Antigravity Will Build These)

3.1 Audio Agent (Recording & Upload Agent)

Responsibilities:

  • Start/stop audio recording
  • Support Portuguese/Spanish/English
  • Handle local caching if offline
  • Upload audio file to processing agent

Inputs: Start recording command
Outputs: Audio file URL or blob

3.2 Transcription Agent

Responsibilities:

  • Convert audio → text
  • Detect language
  • Translate to English if needed
  • Segment by room/task (based on contractor speech)

Inputs: Audio file
Outputs: JSON with: 

{
  "full_transcript": "...",
  "segments": [...],
  "language": "pt" | "es" | "en"
}

3.3 Scope Extraction Agent

Responsibilities:

  • Extract structured scope-of-work
  • Extract materials, quality, measurements, tasks, complexity, timeline hints
  • Infer missing values reasonably
  • Compute recommended profit margin default (40–60%)

Input: Transcript JSON
Output: Structured scope: 

{
  "rooms": [...],
  "materials": [...],
  "labor_requirements": [...],
  "timeline_notes": "...",
  "complexity": "low|medium|high"
}

3.4 Estimate Generation Agent

Responsibilities:

  • Convert structured scope → itemized estimate
  • Calculate:
    • materials cost
    • labor cost
    • timeline
    • GC margin
    • total estimate
    • recommended quote range
  • Use configurable regional rates
  • Return <60s< /li>

Input: Structured scope JSON
Output: 

{
  "items": [...],
  "material_cost": ...,
  "labor_cost": ...,
  "total_cost": ...,
  "margin": 0.4-0.6,
  "recommended_quote": ...,
  "options_good_better_best": {...}
}

3.5 Sales Coach Agent

Responsibilities:

  • Provide pre-estimate prompt questions
  • Provide negotiation scripts for non-native speakers
  • Provide objections-handling suggestions

Input: None or job-type context
Output: List of prompts or scripts

3.6 Estimate Editing Agent

Responsibilities:

  • Allow contractor to edit items, materials, margins
  • Recalculate totals on demand
  • Provide “recompute estimate” option

Input: Edited estimate JSON
Output: Updated estimate JSON

3.7 Presentation Agent (PDF / UI Renderer)

Responsibilities:

  • Generate polished PDF-style representation
  • Branding: V2RBuilds
  • Layout:
    • Logo
    • Room-by-room
    • Totals
    • Options A/B/C
  • Produce shareable link or downloadable file

Input: Final estimate JSON
Output: Rendered PDF-like file or object

3.8 Compliance Agent

Responsibilities:

  • Show dual-consent recording notice
  • Show liability disclaimer
  • Require contractor acknowledgement

Input: None
Output: Boolean on user agreement

5. Workflows (End-to-End)

Workflow A — Generate Estimate

  1. User taps Start Recording → Audio Agent
  2. Stop Recording → Transcription Agent
  3. Transcript → Scope Extraction Agent
  4. Scope → Estimate Generation Agent
  5. Show Estimate Review Screen
  6. User edits → Estimate Editing Agent
  7. User approves → Compliance Agent → Presentation Agent
  8. Show PDF & share options

Workflow B — Negotiation Help

  1. User taps Sales Coach
  2. Sales Coach Agent returns prompts/scripts
  3. Display to user in simple UI

Workflow C — Recalculate Estimate

  1. User adjusts sliders or edits items
  2. Send updated input → Estimate Editing Agent
  3. Return updated totals
  4. Update UI in real time

7. Constraints & Requirements

  • Estimate generation <60 seconds
  • Recording up to 60 minutes
  • Works offline with queued upload
  • Accuracy target: ±15%
  • Bilingual UX (EN/PT/ES)
  • Secure storage + encryption
  • Contractor must approve estimate before sharing
  • V2RBuilds not liable for inaccuracies
